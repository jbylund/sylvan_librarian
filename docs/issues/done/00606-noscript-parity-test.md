# No-JS Parity Test

`noscript_helpers.py` and `createCardHTML()` in `app.js` must produce equivalent card HTML, but there is no test that enforces this. When one renderer changes, the other has to be updated manually — and if it is missed, the no-JS path silently shows something wrong.

## What to build

A parity test that renders a fixed set of representative cards through both renderers and asserts HTML equality (or structural equivalence). Candidates:

- A card with a mana cost (exercises mana symbol conversion)
- A card with oracle text over 200 characters containing a mana symbol near the cutoff
- A creature with power/toughness
- A card with no oracle text, no mana cost (edge case)

The Python side is straightforward: call `create_card_html(card, index)` directly.

**Update 2026-07-03: the JS-side blocker is solved and a template exists.** PR #595
added a Jest suite (`api/static/app.test.js`) running `app.js` code under Node, and
PR #597 established the shared-fixture parity pattern this test should copy:
a fixture in `api/static/fixtures/` (see `balance_queries.json`) consumed by both
the Jest suite and a pytest (`test_balance_parity.py`). Remaining work is one
fixture of the representative cards above + a test on each side; the only open
question is whether `createCardHTML()` (a method on the app class) needs light
extraction or a jsdom shim beyond what the already-ported functions required.

**Resolved 2026-07-03.** The renderers had drifted in four ways; three were converged
(Python gained the name + mana cost + oracle alt text and the card-detail-page link
wrap; both sides now back up before an unclosed `{` when truncating oracle text at
200 chars; Python's `escape_html` dropped `'`-escaping to match JS; JS alt-text
truncation counts code points instead of UTF-16 units, so emoji mana symbols can't be
split and the 300-char cutoff matches Python slicing). The fourth —
`fetchpriority`/`loading` logic — was deliberately left divergent (revisit later) and
is stripped by the test's normalizer, along with inter-tag whitespace. The contract
lives in `api/static/fixtures/card_html_cases.json` (6 cases, regenerated by
`scripts/generate_card_html_fixture.py`), enforced by
`api/tests/test_noscript_parity.py` and a describe block in `api/static/app.test.js`.

## Why it matters

Silent divergence gets filed by screen reader and CLI browser users, not typical users, so it tends to stay unfixed longer than it should. A parity test would catch any drift immediately.
